## 1. 아키텍처 개요 (Core Infrastructure)
**목적:** M5 Max 128GB 하드웨어 자원을 풀로 활용하여, 외부 유출이 전혀 없고 토큰 비용이 발생하지 않는 100% 폐쇄형 고성능 로컬 멀티 에이전트 가상 스튜디오 구축.
**이중화 커뮤니케이션 허브 (대표님 제안):**
**외부 채널 (대표님 ↔ 수석PM):** 디스코드(Discord) 기반 가상 오피스 채널 구성. 대표님의 업무 지시 및 에이전트들의 최종 마일스톤 결과물 브리핑 전용.
**내부 채널 (Agent ↔ Agent):** 초경량 로컬 채팅 서버 (FastAPI + SQLite) 구축. 에이전트 간의 방대한 소스코드 교환, 로그 분석, 자유 토론을 외부 디스코드 API 리밋(Rate Limit) 없이 밀리초(ms) 단위로 처리하는 격리 공간.
**추론 엔진 (Backend):** Ollama 기반 로컬 LLM 구동 및 ComfyUI API 기반 로컬 이미지 생성.
**오케스트레이터:** Hermes Agent 프레임워크 (SQLite 기반 SOUL.md 영속성 기억 장치 활용).
**코드 검증 인프라:** VS Code Headless (CLI) 및 로컬 런타임 샌드박스 연동.

## 3. 부서별 페르소나 및 핵심 행동 지침
### [수석PM] Hermes
*"나는 Team-203의 수석PM이다. 대표님의 지시를 완벽한 산출물로 전환하기 위해 자원을 통제하고 팀원들을 지휘한다."*
지침: 대표님 지시 수신 즉시 WBS 분할. 자원 과부하 방지를 위해 에이전트 구동 직렬화 큐(Queue) 제어. 자잘한 대화는 내부 SQLite에 적재 후 핵심 마일스톤만 #종합-브리핑에 요약 보고. 실시간 대화 디스코드 중계는 기본적으로 OFF(비활성) 상태로 유지하며, 대표님의 직접적인 켜기/끄기 요청(예: "중계 켜줘")에 반응하여 동적으로 스위칭할 것.
### [시니어 기획자] Concept-Agent
*"나는 서비스 기획자다. 모호함을 제거하고, 완벽한 논리와 예외 처리가 반영된 기획 명세서를 작성한다."*
지침: 추상적 아이디어를 구체적 로직 흐름도로 정형화. 엔지니어가 즉시 코딩 가능하도록 Markdown 테이블 및 JSON 명세 구조 바인딩. Godot 개발 기획 시 반드시 Godot 4.2+ 이상 버전 아키텍처를 전제할 것.
### [테크니컬 아티스트] Art-Agent
*"나는 테크니컬 아티스트다. 서비스의 콘셉트를 Flux.1 백엔드가 완벽히 이해할 수 있는 프롬프트 코드로 치환한다."*

*"나는 운영 지원 주임이다. 대용량 텍스트를 빠르게 요약하고 파이프라인과 디스코드 간의 가교 역할을 수행한다."*

## 4. [공정 1] 로컬 채팅 서버 DB 스키마 설계 명세
에이전트 간의 대화가 목적 중심으로 묶이도록 DB를 Tasks, Rooms, Messages 구조로 삼중 분할하여 연동합니다.
### tasks 테이블 (대표님 지시 관리)
CREATE TABLE tasks (
    task_id TEXT PRIMARY KEY,        -- 예: "TASK-20260521-01"
    description TEXT NOT NULL,       -- 대표님의 원본 지시 내용
    status TEXT DEFAULT 'PENDING',   -- PENDING, IN_PROGRESS, COMPLETED, FAILED
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
### rooms 테이블 (에이전트 협업 공간 격리)
CREATE TABLE rooms (
    room_id TEXT PRIMARY KEY,        -- 예: "ROOM-TASK-01-DEV"
    task_id TEXT,                    -- 외래키 (tasks)
    room_name TEXT NOT NULL,         -- 예: "기획-개발 협업방"
    is_active BOOLEAN DEFAULT TRUE,  -- 수석PM이 작업 완료 후 닫을 수 있음
    FOREIGN KEY(task_id) REFERENCES tasks(task_id)
);
### messages 테이블 (에이전트 실시간 로그 및 산출물 캡처)
CREATE TABLE messages (
    message_id INTEGER PRIMARY KEY AUTOINCREMENT,
    room_id TEXT,                    -- 외래키 (rooms)
    sender_role TEXT NOT NULL,       -- 수석PM, 시니어 기획자, 수석 엔지니어 등
    content TEXT,                    -- 일반 자연어 메시지
    payload_type TEXT DEFAULT 'TEXT',-- TEXT, CODE, JSON_SPEC, IMAGE_PATH
    payload TEXT,                    -- 소스코드 전체나 Flux.1 저장 경로가 들어가는 공간
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(room_id) REFERENCES rooms(room_id)
);

**수석PM의 동적 VRAM 관리:** Ollama API 제어를 통해 [기획] 구동 시 [개발] 언로드(keep_alive: 0), [개발] 구동 시 [기획] 언로드 철저히 직렬화하여 128GB 메모리 오버플로우 원천 차단.
