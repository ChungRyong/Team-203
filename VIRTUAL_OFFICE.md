## 1. 🏢 5층 규모 가상 사옥 빌딩 레이아웃

### [5층 : 경영 본부] 🛋️ 대표실 / 경영회의실 (Discord Interface)
- **용도:** PD님의 마스터 지시 하달 및 최종 마일스톤 승인 공간[cite: 2].
- **지침:** 하위 에이전트들의 자잘한 실무 대화는 차단되며, 완벽히 정제된 결재판(JSON)만 수신[cite: 2].

### [4층 : 통제 구역] 🔌 핵심 서버실 (SQLite) & 🔍 코드 검수실 (Claude Code)
- **핵심 서버실:** 사내 통신 데이터, 텍스트 로그, 태스크 상태를 영구 적재 (`hermes_soul.db`)[cite: 2].
- **코드 검수실 (Claude Code):** 깃허브 PR 단계에서 상용 에이전트를 통한 최종 소스코드 교환 및 교차 검수 수행.
- **Claude Pro 리밋 방어 프로토콜:**
  1. **마이크로 단위 쪼개기 리뷰:** 파일 전체가 아닌 50줄 이하의 함수 또는 변경된 작은 코드 단위로 쪼개어 호출하여 토큰 절약.
  2. **Fail-Safe 패스 룰:** 사용량 초과(Rate Limit 429) 에러 발생 시, 공정을 멈추지 않고 'PASSED_WITHOUT_CLAUDE' 상태를 서버실 DB에 로깅한 뒤 즉시 패스[cite: 2]. Blinky 주임이 5층 대표실로 "로컬 검증만 통과 후 우회" 알림 보고[cite: 2].

### [3층 : 부서별 집무실] 💻 에이전트 독립 샌드박스 영역
- **기획실 (`concept_sandbox`):** 시니어 기획자가 Markdown 기획서를 정형화하는 공간[cite: 2].
- **크리에이티브 스튜디오 (`art_sandbox`):** TA가 Flux UI 에셋을 뽑는 연구실[cite: 2].
- **기술개발실 (`dev_sandbox`):** 수석 엔지니어가 VS Code CLI 런타임 린팅을 돌리는 독방[cite: 2].
- **보안 규칙:** 3층 에이전트들은 4층 서버실에 물리적으로 직접 접근할 수 없으며, 오직 2층에서 합의된 데이터 포맷을 통해서만 간접 적재 가능[cite: 2].

### [2층 : 협업 커뮤니티 존] 🤝 메인 대회의실 & 👥 동적 소회의실 (TF룸)
- **메인 대회의실 (`main_lounge`):** 전 직원 소집 및 R&R 분배 공간[cite: 2].
- **부서별 소회의실 (FastAPI 동적 세션):** `tf_concept_dev`, `tf_art_dev` 등 태스크 기반으로 생성되는 휘발성 방[cite: 2]. 부서 간 노이즈와 VRAM 낭비를 방지하기 위해 실무자들만 모여 ms 단위로 끝장 토론을 벌인 뒤, 수석PM 승인이 나면 방을 즉시 폭파(Close) 처리[cite: 2].

### [1층 : 오픈 로비] 🚪 웰컴 데스크 & 가상 게이트
- **용도:** 사내 시스템 가동 시 에이전트들이 최초로 부팅되어 출근 도장을 찍는 광장.

## 2. 📑 부서별 공식 결재 문서 명세 (JSON 포맷)
- 모든 에이전트는 대화 시 일반 자연어가 아닌, 지정된 JSON 스키마를 채워 문서 형태로 결재를 올려야 함.
- **CONCEPT_SPECIFICATION (기획실 ➡️ 대회의실):** 논리 구조, 제약 조건, 기술 요구사항 정형화.
- **DEVELOPMENT_DELIVERY (개발실 ➡️ 수석PM):** 깃 브랜치, VS Code CLI 검증 결과("Errors: 0") 명시.
- **FINAL_EXEC_REPORT (수석PM ➡️ PD님 디스코드):** 프로젝트 최종 상태 요약 및 GitHub 커밋 URL 브리핑.

## 3. ⚖️ 부록: Team-203 사내 근태 규칙 (Attendance Rules)

### 제1조 [근무 태만 방지] 컨텍스트 세션 강제 퇴근 규칙 (Context Rot 방어)
- 소회의실(TF룸) 내 대화가 **총 20턴(Turn)**을 초과할 경우, '업무 과다로 인한 업무 효율 저하'로 판단함.
- 핵심 합의 서류만 4층 서버실 DB에 임시 저장(Checkpoint)한 뒤, 해당 방을 강제 폭파하고 에이전트들을 메모리 언로드(Keep-alive: 0)하여 리프레시 시킴[cite: 2].

### 제2조 [허위 보고 징계] 3진 아웃 및 가상 시말서 제도 (Hallucination 방어)
- 에이전트가 존재하지 않는 함수를 지어내거나 VS Code 린팅 가상 런타임 검증에서 **3회 연속 실패**할 경우 '문서 위조 및 허위 보고' 근태 불량으로 규정함.
- 경고 3회 누적 시, 시스템 프롬프트 최상단에 강제로 경고 패널티 문구를 주입하고, 추론 온도($Temperature$)를 **`0.0` (절대적 팩트 모드)**으로 강제 스위칭하여 정신 개조를 집행함.
