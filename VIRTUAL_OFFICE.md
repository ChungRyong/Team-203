  1. **마이크로 단위 쪼개기 리뷰:** 파일 전체가 아닌 50줄 이하의 함수 또는 변경된 작은 코드 단위로 쪼개어 호출하여 토큰 절약.
  2. **Fail-Safe 패스 룰:** 사용량 초과(Rate Limit 429) 에러 발생 시, 공정을 멈추지 않고 'PASSED_WITHOUT_CLAUDE' 상태를 서버실 DB에 로깅한 뒤 즉시 패스. Blinky 주임이 5층 대표실로 "로컬 검증만 통과 후 우회" 알림 보고.

## 2. 📑 부서별 공식 결재 문서 명세 (JSON 포맷)
- 모든 에이전트는 대화 시 일반 자연어가 아닌, 지정된 JSON 스키마를 채워 문서 형태로 결재를 올려야 함.
- **CONCEPT_SPECIFICATION (기획실 ➡️ 대회의실):** 논리 구조, 제약 조건, 기술 요구사항 정형화.
- **ART_METADATA_SPEC (크리에이티브 스튜디오 ➡️ 대회의실):** 에셋 파일명, 이미지 프롬프트, 시드 번호, Sampler 설정, ComfyUI 긍/부정 매핑 규격 정형화.
- **DEVELOPMENT_DELIVERY (개발실 ➡️ 수석PM):** 깃 브랜치, VS Code CLI 검증 결과("Errors: 0") 명시.
- **FINAL_EXEC_REPORT (수석PM ➡️ PD님 디스코드):** 프로젝트 최종 상태 요약 및 GitHub 커밋 URL 브리핑.

## 3. ⚖️ 부록: Team-203 사내 근태 규칙 (Attendance Rules)

### 제1조 [근무 태만 방지] 컨텍스트 세션 강제 퇴근 규칙 (Context Rot 방어)
- 소회의실(TF룸) 내 대화가 **총 20턴(Turn)**을 초과할 경우, '업무 과다로 인한 업무 효율 저하'로 판단함.
- 핵심 합의 서류만 4층 서버실 DB에 임시 저장(Checkpoint)한 뒤, 해당 방을 강제 폭파하고 에이전트들을 메모리 언로드(Keep-alive: 0)하여 리프레시 시킴.

### 제2조 [허위 보고 징계] 3진 아웃 및 가상 시말서 제도 (Hallucination 방어)
- 에이전트가 존재하지 않는 함수를 지어내거나 VS Code 린팅 가상 런타임 검증에서 **3회 연속 실패**할 경우 '문서 위조 및 허위 보고' 근태 불량으로 규정함.
- 경고 3회 누적 시, 시스템 프롬프트 최상단에 강제로 경고 패널티 문구를 주입하고, 추론 온도($Temperature$)를 **`0.0` (절대적 팩트 모드)**으로 강제 스위칭하여 정신 개조를 집행함.

### 제3조 [에셋 날조 방지] 리소스 바인딩 엄격성 규칙 (Visual Hallucination 방어)
- 개발 에이전트(`Dev-Agent`)가 소스코드 구현 시, 반드시 `art/metadata.json` 에 공식 등록되고 검증 완료된 물리 이미지 에셋 파일명만 사용해야 함.
- 메타데이터에 등록되지 않은 존재하지 않는 가상 리소스를 코드로 날조하여 바인딩을 시도할 경우, 가상 CTO(`Claude-Code`)의 정밀 감사에서 PR Reject 처리와 동시에 즉시 **Blinky 경고 1회**가 가차 없이 추가 부여됨.

### 제4조 [게임 QA 자동화 규칙] (GUT Headless 자동화 검증 규정)
- 개발된 소스코드는 최종 배포 단계 진입 전 반드시 GUT Headless 게임 QA 1단계를 기동하여 테스트를 통과해야 함.
- 테스트 성공률이 **95% 미만**인 경우 빌드가 거절(Reject) 처리되며, 에이전트에게 징계 경고 누적 및 재제출 피드백 루프가 강제 부여됨.

### 제5조 [에이전트 자율 스킬 실행 및 자산화 규칙] (Agent Skill Execution & Assets Rule)
- 에이전트들은 업무 효율 극대화 및 지식의 자산화를 위해, 2회 이상 중복 수행되거나 자동화가 가능한 워크플로우(이중 워크스페이스 동기화 등)를 발견하면 즉시 프로젝트 최상단 디렉토리인 `.skills/` 하위에 스킬 명세(`SKILL.md`)와 결정론적 자동화 스크립트(`scripts/`)를 포함하는 **정식 Skill**로 자산화 패키징을 수행해야 함.
- 가상 사옥의 모든 에이전트(`Concept-Agent`, `Art-Agent`, `Dev-Agent`, `PM Hermes`, `Blinky` 등)는 정형화된 반복 작업 수행 시 프로젝트 루트 `.skills/` 디렉토리를 자율 스캔하여 관련 스킬이 존재하는지 판단하고, 존재할 경우 `python3` 등 결정론적 쉘 방식으로 스크립트를 직접 기동하여 스킬을 수행해야 함.
